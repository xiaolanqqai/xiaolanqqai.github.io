关于

MiniServer Panel — 极简服务器管理面板
纯浏览器端 SSH 客户端，通过服务器上的轻量 bridge 程序连接。
静态页面部署在 GitHub Pages，手机/电脑浏览器均可使用。
版本 3.4.0

首次使用引导

本面板通过服务器上的轻量 C 桥接程序（bridge）连接 SSH。使用前请先在服务器上完成安装，整个过程约 1 分钟。

推荐使用一键安装脚本，自动完成下载、编译、运行。

★

方式一：一键安装（推荐）

在服务器上执行一行命令，自动完成：下载源码 → 安装 gcc → 编译 → 启动。

curl 一键安装 
curl -fsSL https://raw.githubusercontent.com/xiaolanqqai/xiaolanqqai.github.io/master/index/Tool/web/install.sh | bash

或使用 wget：

wget 一键安装 
wget -O - https://raw.githubusercontent.com/xiaolanqqai/xiaolanqqai.github.io/master/index/Tool/web/install.sh | bash

脚本运行完会自动启动 bridge，无需手动配置 Token。

方式二：手动分步安装

如需手动控制每个步骤，可按以下顺序执行：

1. 登录服务器

终端 
ssh root@你的服务器IP

2. 下载源码

wget 
wget -O bridge.c https://raw.githubusercontent.com/xiaolanqqai/xiaolanqqai.github.io/master/index/Tool/web/bridge.c

3. 安装 gcc（如未安装）

apt 
apt update && apt install -y gcc

4. 编译

gcc 
gcc -O2 -o bridge bridge.c

5. 运行

前台运行 
./bridge
[bridge] listening on :8022  token=ssh (固定值，前端自动填写)

6. 后台常驻（可选）

nohup 
nohup ./bridge > bridge.log 2>&1 &

7. 放行防火墙端口

ufw 
ufw allow 8022/tcp
✓

回到面板连接

安装完成！点击右上角连接按钮，填写以下信息：

服务器地址你的服务器 IP
Bridge 端口8022
SSH 主机127.0.0.1
SSH 端口22
用户名 / 密码SSH 登录凭据

常见问题

Q: 连接提示"WebSocket 连接失败"？

A: 请确认 bridge 已在服务器上运行，且防火墙已放行端口。

Q: 需要配置 Token 吗？

A: v3.4 起 Token 固定为 "ssh"，前端自动填写，用户无需关心。

Q: 内存占用多少？

A: bridge 稳定占用 < 0.5MB RAM，适配 100MB 内存的低配 VPS。

Q: 手机能用吗？

A: 可以，任何现代浏览器均可使用，无需安装任何插件。
